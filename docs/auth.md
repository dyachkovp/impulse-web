# Авторизация: вход через Сбер ID

Единственный способ авторизации в Impulse — **Сбер ID**. Отдельной регистрации нет: первый успешный вход через Сбер ID создаёт аккаунт пользователя.

## 1. Экраны

### 1.1. `/` — экран входа Impulse

Публичная страница, с которой начинается любая пользовательская сессия. Содержит:

- Крупный логотип «Импульс» (по центру).
- Подзаголовок курсивом: *«AI-driven платформа проектно-ориентированного обучения от Сбера»*.
- Чекбокс согласия:
  - Текст: «Я даю своё согласие на **обработку персональных данных** и принимаю условия **Пользовательского соглашения**».
  - Оба выделения — ссылки (в прототипе некликабельны или ведут на внешний URL).
- Кнопка **«Войти по Сбер ID»** (акцентная, зелёная).
- Сноска в футере: ссылка «Политика конфиденциальности» (некликабельна в прототипе).

### 1.2. `/sber-id` — имитация страницы Сбер ID

Это **не часть Impulse** — отдельный сервис авторизации. В прототипе делаем скелетон, визуально отличный от основного приложения (зелёный фирменный фон), чтобы было понятно, что пользователь покинул Impulse.

Содержит минимум:
- Заголовок: *«Это вы, Павел Д.?»*.
- Аватар-плейсхолдер с инициалами.
- Две кнопки:
  - **«Продолжить»** — подтверждает вход (зелёная, акцентная).
  - **«Это не я»** — отказ (нейтральная).
- Короткая приписка об условиях оферты (можно опустить в прототипе).

Реальной аутентификации нет: кнопки имитируют редирект обратно в Impulse.

## 2. Правила

### 2.1. Валидация на экране входа

Кнопка «Войти по Сбер ID» **неактивна**, пока пользователь не поставил галочку согласия. После клика по чекбоксу — становится кликабельной.

### 2.2. Переходы

| Действие | Результат |
|---|---|
| Пользователь поставил галочку + нажал «Войти по Сбер ID» | Переход на `/sber-id` |
| На `/sber-id` нажал «Продолжить» | Переход в приложение (`/dashboard`) |
| На `/sber-id` нажал «Это не я» | Возврат на `/` (галочка сбрасывается) |
| Попытка открыть защищённый маршрут без авторизации | Редирект на `/` |

### 2.3. Состояние авторизации

В прототипе признак авторизации хранится в памяти сессии (например, булев флаг). При перезагрузке страницы пользователь считается неавторизованным и возвращается на `/`.

## 3. Что **не** реализовано в прототипе

- Реальная интеграция со Сбер ID (OAuth-флоу, редиректы, exchange кода на токен).
- Отдельная регистрация, восстановление доступа.
- Выход из аккаунта с подтверждением.
- Роли помимо Студента (куратор, администратор).
- Онбординг после первого входа — см. [student-actions.md §1.1](student-actions.md#11-знакомство-с-платформой).
