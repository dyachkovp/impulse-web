# Онбординг нового пользователя

После успешной авторизации через Сбер ID ([auth.md](auth.md)) пользователь попадает в одну из двух веток:

- **Зарегистрированный** (бывал на платформе раньше) → сразу `/dashboard`.
- **Новый** (первый вход) → онбординг.

Онбординг — единоразовый флоу для новых студентов: собираем минимальный профиль, чтобы рекомендовать релевантные проекты и открыть возможность подачи заявок.

## 1. Как определяется новый пользователь

В прототипе факт регистрации хранится в памяти сессии. Любой первый вход считается регистрацией → запускается онбординг. При повторном заходе (без перезагрузки страницы) онбординг уже не показывается.

В продакшене — признак существующей учётной записи на бэкенде, привязанной к Сбер ID.

## 2. Экраны

### 2.1. `/onboarding/about` — «Расскажите о себе»

Первый шаг. Собирает базовый профиль студента.

**Хедер** (общий для всех шагов онбординга): логотип «Импульс» слева, кнопка «Выйти» справа.

**Поля:**

| Поле | Тип | Источник / правила |
|---|---|---|
| **Имя** | текст, readonly | Берётся из Сбер ID. Нельзя редактировать. Под полем — галочка-индикатор «загружено из Сбер ID» |
| **Фамилия** | текст, readonly | Берётся из Сбер ID. Нельзя редактировать. Индикатор тот же |
| **Регион** | select (поиск) | **Обязательное**. Полный список субъектов РФ (см. `src/data/regions.ts`) |
| **Учебное заведение** | select (поиск) | **Обязательное**. Список зависит от выбранного региона (`src/data/universities.ts`). **Заблокировано** пока регион не выбран. При смене региона — сбрасывается |
| **Чем увлекаешься?** | textarea | Необязательное. Плейсхолдер с примером |

**Подсказка под readonly-полями:** единое сообщение «Данные загружены из Сбер ID» в виде info-плашки над полями.

**Кнопка «Продолжить»** — активна, когда заполнены регион и учебное заведение. Клик → `/onboarding/specialization`.

### 2.2. `/onboarding/specialization` — «Выбери свою специализацию»

Второй шаг. Студент выбирает профессиональную роль.

**Ограничение онбординга:** можно выбрать **ровно одну** специализацию из 7 ([см. data-model.md §4.1](data-model.md#41-специализации-7-шт)). Это обязательно. Позже в профиле можно добавить другие.

**Список вариантов** отображается вертикально — карточки с названием специализации. Выбранная подсвечивается акцентом.

**Кнопка «Продолжить»** — активна, когда выбрана специализация. Клик → `/onboarding/assessment`.

### 2.3. `/onboarding/assessment` — «Оцени свои компетенции»

Третий шаг. Студент оценивает навыки по выбранной специализации.

**Структура:** заголовок «Оцени свои компетенции» и подзаголовок с названием специализации. Под ними — прогресс «Заполнено: X из N» со шкалой.

**Группы навыков** отображаются аккордеоном. Для каждой группы:

- **Хедер группы** — название + шеврон. Справа:
  - `★ 2,8` (средняя оценка с одним знаком) — когда все навыки группы оценены;
  - `3/7` — когда оценена часть навыков;
  - кнопка `☆ Оценить` — когда в группе нет оценок и она свёрнута.
- **Тело группы** (развёрнутое) — список навыков. Каждый навык: название слева, 5 звёзд справа и текстовый лейбл.

По умолчанию первая группа раскрыта. Одновременно раскрыта не более одной группы.

**Шкала оценки** (источник правды — `RATING_LABELS` в `src/data/skills.ts`):

| Звёзды | Лейбл |
|---|---|
| 1 | Нет опыта |
| 2 | Начальный |
| 3 | Средний |
| 4 | Продвинутый |
| 5 | Экспертный |

**Взаимодействие со звёздами:**

- Клик по N-й звезде ставит оценку N (все предыдущие звёзды заполняются).
- Повторный клик по текущей оценке — снимает её (как в мобильном приложении).

**Кнопки внизу:**

- `Сохранить и продолжить` — сохраняет оценки и переходит на `/dashboard`.
- `Оценить позже` — пропускает шаг и переходит на `/dashboard`. Уже поставленные оценки сохраняются в сессии.

Обе кнопки завершают онбординг (`completeOnboarding`). Оценки живут в памяти сессии (`session.assessment`) в форме `specialization → skill → 1..5`.

## 3. Флоу

```
/sber-id  ─→  первый вход?
                │
       да ──────┴────── нет
       │                │
       ▼                ▼
/onboarding/about    /dashboard
       │
       ▼
/onboarding/specialization
       │
       ▼
/onboarding/assessment
       │
       ▼
/dashboard
```

## 4. Правила

- При любом шаге пользователь может нажать «Выйти» в хедере — возврат на `/` (прогресс онбординга теряется).
- Данные онбординга хранятся в памяти сессии до завершения. После завершения — как профиль.
- Пользователь не может попасть на `/dashboard`, пока не прошёл обязательные шаги онбординга (регион, учебное заведение, одна специализация).
- Возврат назад между шагами онбординга допустим (например, через браузерную кнопку back). Данные на предыдущем шаге сохраняются.

## 5. Что не реализовано в прототипе

- Реальное получение ФИО из Сбер ID (в прототипе — захардкожено «Павел Дьячков»).
- Самооценка из профиля (пока доступна только как шаг онбординга, но фича задумана как многоразовая).
- Валидация на бэкенде (дубли, запрещённые символы, и т.д.).
- Возможность «пропустить» онбординг — шаги «Расскажите о себе» и «Специализация» обязательны; самооценку можно отложить через «Оценить позже».
