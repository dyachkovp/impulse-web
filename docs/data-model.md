# Модель данных: гипотезы и проекты

Документ описывает структуру данных, с которыми работает студент, и связи между сущностями. Поля названы так же, как в исходниках прототипа.

## 1. Гипотеза (Hypothesis)

Гипотеза — это исследовательская или бизнесовая задача, сформулированная заказчиком. Она задаёт контекст, а конкретная реализация происходит уже внутри проектов.

### Поля

| Поле | Тип | Описание |
|---|---|---|
| `id` | number | Уникальный идентификатор |
| `title` | string | Название гипотезы |
| `shortDescription` | string | Короткое описание для списка |
| `projects` | Project[ ] | Встроенный список проектов, реализующих гипотезу (облегчённые карточки для отображения в списке) |

### Бриф гипотезы

В детальной карточке отображаются шесть смысловых блоков. В прототипе они одинаковые для всех гипотез, но по смыслу это контент, который задаёт бизнес-заказчик:

- **`goal`** — проблема и цель: что именно нужно решить и зачем.
- **`context`** — бизнес-контекст, боли пользователей, ограничения.
- **`tasks`** — ключевые рабочие задачи (обычно 4 пункта).
- **`resources`** — доступные инструменты, датасеты, поддержка.
- **`criteria`** — критерии успеха (обычно 5 пунктов).
- **`result`** — ожидаемый результат и формат отчётности.

### Состояния (по наличию проектов)

Явного поля «статус» у гипотезы нет, но в UI используется фильтрация:
- **«С проектами»** — `projects.length > 0`;
- **«Без проектов»** — `projects.length === 0`;
- **«Все»** — без фильтра.

## 2. Проект (Project)

Проект — это конкретная команда, которая берётся решать гипотезу за фиксированный период. У проекта есть жизненный цикл, команда, вакансии, задачи и материалы.

### Поля

| Поле | Тип | Описание |
|---|---|---|
| `id` | string | Уникальный идентификатор (`"p1"`, `"p-2-1"`) |
| `hypothesisId` | number | Ссылка на родительскую гипотезу |
| `hypothesisTitle` | string | Название гипотезы (денормализовано для оффлайн-UX) |
| `title` | string | Название проекта |
| `description` | string | Полное описание |
| `shortDescription` | string | Короткое описание для списков |
| `status` | enum | Текущий статус проекта (см. ниже) |
| `curator` | string | ФИО куратора |
| `startDate` | string | Дата старта в формате `DD.MM.YYYY` |
| `endDate` | string | Дата завершения в формате `DD.MM.YYYY` |
| `tags` | string[ ] | Требуемые специализации (`"Backend"`, `"Frontend"`, `"System Analyst"` и т.п.) |
| `members` | Member[ ] | Текущий состав команды |
| `vacancies` | Vacancy[ ] | Открытые позиции |
| `tasks` | Task[ ] | Задачи проекта (видны в статусе «Работа над прототипом») |
| `resources` | Resource[ ] | Материалы и ссылки |

### Статусы проекта

- **«Формирование команды»** — проект набирает участников, открыты вакансии, задачи и материалы ещё не показываются.
- **«Работа над прототипом»** — команда собрана, ведётся работа, доступны задачи и ресурсы.

### Участник команды (Member)

| Поле | Тип | Описание |
|---|---|---|
| `name` | string | ФИО |
| `specialization` | string | Роль в команде |
| `university` | string | Вуз |
| `isYou` | boolean? | `true`, если это текущий пользователь |

### Вакансия (Vacancy)

| Поле | Тип | Описание |
|---|---|---|
| `id` | string | Идентификатор вакансии |
| `specialization` | string | Требуемая специализация |
| `count` | number | Сколько мест открыто |
| `initialState` | enum | Состояние вакансии для текущего пользователя |
| `appliedAt` | timestamp? | Когда студент подал заявку |
| `invitedAt` | timestamp? | Когда студента пригласили |

Состояния вакансии (жизненный цикл с точки зрения студента):

```
open  ──(студент подал заявку)──► submitted ──► applied
  │
  └──(куратор пригласил)────────► invited ──► (принято/отклонено)
```

### Задача (Task) — видна только в статусе «Работа над прототипом»

| Поле | Тип | Описание |
|---|---|---|
| `title` | string | Название задачи |
| `status` | enum | `"Готово"` / `"В работе"` / `"Не начато"` |
| `assignee` | string \| null | Исполнитель или `null`, если не назначен |

### Ресурс (Resource)

| Поле | Тип | Описание |
|---|---|---|
| `title` | string | Название материала |
| `url` | string | Ссылка (в прототипе некликабельна) |

## 3. Связи между сущностями

```
Hypothesis (1) ──────────── (0..N) Project
    id                              hypothesisId
                                    hypothesisTitle (денормализация)

Project (1) ─────── (N) Member       (состав команды)
Project (1) ─────── (N) Vacancy      (открытые позиции)
Project (1) ─────── (N) Task         (только в статусе «Работа над прототипом»)
Project (1) ─────── (N) Resource     (материалы)

Student (1) ─────── (N) Specialization  (выбранные роли)
Specialization (1) ─── (N) SkillGroup ─── (N) Skill
Skill (1) ─────── (0..1) Rating (1..5)    (самооценка, в памяти сессии)
```

**Ключевые правила связей:**

- Проект всегда принадлежит ровно одной гипотезе (`hypothesisId`).
- Гипотеза может не иметь ни одного проекта.
- В списке `Hypothesis.projects` хранятся облегчённые карточки для ленты; полная информация — в отдельной коллекции проектов.
- Студент может быть участником (`Member`) одновременно нескольких проектов.
- `Vacancy.specialization` должна совпадать с одной из специализаций в профиле, чтобы студент мог подать заявку.

## 4. Справочники

### 4.1. Специализации (7 шт.)

- Backend
- Frontend
- QA
- Designer
- Product Owner
- System Analyst
- Business Analyst

Одинаково используются в профиле студента, тегах проектов, вакансиях и структуре самооценки. Источник правды — `SKILLS_STRUCTURE` в `src/data/skills.ts`.

### 4.2. Структура самооценки (SKILLS_STRUCTURE)

```
Специализация (7)
  └── Группа навыков (4–7 на специализацию)
        └── Навык (2–12 на группу)
              └── Оценка: 1..5 | null
```

Шкала оценки:

| Значение | Смысл |
|---|---|
| 1 | Нет опыта |
| 2 | Начальный |
| 3 | Средний |
| 4 | Продвинутый |
| 5 | Экспертный |

Оценки хранятся в памяти сессии (`specialization → skill → rating`) и сбрасываются при перезагрузке страницы.

### 4.3. Профиль студента

| Поле | Тип | Источник |
|---|---|---|
| `firstName`, `lastName` | string | Вводится пользователем |
| `email` | string | Вводится пользователем |
| `region` | string | Выбирается из захардкоженного списка регионов |
| `university` | string | Выбирается из захардкоженного списка вузов или вводится вручную |
| `about` | string | Свободное текстовое описание |
| `specializations` | string[ ] | Выбирается из 7 специализаций |

## 5. Объём данных в прототипе

- 15 гипотез;
- 6 проектов (`p1`–`p6`);
- 3–8 участников в активных проектах;
- 2–3 вакансии на проект;
- ~4 задачи на активный проект;
- 7 специализаций × 4–7 групп навыков × 2–12 навыков.

## 6. Технические замечания

- **Денормализация.** `Project.hypothesisTitle` дублирует `Hypothesis.title` — чтобы отображать список проектов без дополнительного запроса.
- **Встроенные списки.** Внутри гипотезы лежит облегчённая версия проектов для ленты; полный проект берётся из отдельной коллекции при переходе в детальную карточку.
- **Сессионное состояние.** Переопределения статусов вакансий, флаг участника и прогресс онбординга живут в памяти вкладки.
- **Отсутствие бэкенда.** Все данные захардкожены или хранятся в памяти; аутентификация и синхронизация не реализованы.
