# Глоссарий

Термины, которые используются в приложении Impulse и в этой документации. Отсортированы по алфавиту (сначала русские, потом английские).

---

## Бизнес-сущности

### Гипотеза (Hypothesis)
Исследовательская или бизнесовая задача от заказчика: описание проблемы, контекста, ожидаемого результата и критериев успеха. Сама по себе гипотеза ничего не реализует — она служит «рамкой» для одного или нескольких проектов. См. [data-model.md §1](data-model.md#1-гипотеза-hypothesis).

### Проект (Project)
Конкретная команда и реализация, которая берётся решать гипотезу за фиксированный период. У проекта есть жизненный цикл, состав команды, вакансии, задачи и материалы. Всегда связан ровно с одной гипотезой через `hypothesisId`. См. [data-model.md §2](data-model.md#2-проект-project).

### Бриф гипотезы
Шесть смысловых блоков в детальной карточке гипотезы: `goal`, `context`, `tasks`, `resources`, `criteria`, `result`. Задаёт содержательный контекст для команд, которые будут её реализовывать.

### Команда (Team)
Совокупность участников (`members`) одного проекта. Формируется через подачу заявок на вакансии и их одобрение. В прототипе состав команды задан заранее.

### Участник (Member)
Человек, который уже входит в команду проекта. Описывается именем, специализацией и вузом. Если участник — это текущий пользователь, в данных появляется флаг `isYou`.

### Вакансия (Vacancy)
Открытая позиция в команде проекта: какая специализация нужна и сколько мест. Имеет состояние с точки зрения конкретного студента: `open`, `submitted`, `applied`, `invited`. См. [data-model.md §2](data-model.md#вакансия-vacancy).

### Заявка (Application)
Действие студента: «хочу занять вакансию». В прототипе заявка не является отдельной сущностью — она выражается через переход вакансии в состояние `submitted` / `applied`.

### Приглашение (Invite)
Обратный сценарий: куратор предлагает студенту присоединиться к проекту. Отражается состоянием вакансии `invited` и полем `invitedAt`.

### Задача (Task)
Рабочий пункт внутри проекта: название, статус (`Готово` / `В работе` / `Не начато`) и назначенный исполнитель. Видна студенту только в статусе проекта «Работа над прототипом».

### Ресурс / Материал (Resource)
Ссылка на внешний материал проекта: прототип, датасет, документ. В прототипе ссылки некликабельны.

### Куратор (Curator)
Ответственный за проект со стороны организаторов / заказчика. В текущей версии — просто строковое поле `curator` на карточке проекта; отдельной роли в UI у куратора нет.

### Специализация (Specialization)
Профессиональная роль: Backend, Frontend, QA, System Analyst, Product Owner, Designer, Business Analyst. Используется в трёх местах: в профиле студента, в тегах проекта и в вакансиях. См. [data-model.md §4.1](data-model.md#41-специализации-7-шт).

### Самооценка (Self-Assessment)
Процесс, в котором студент оценивает свои навыки по шкале 1–5 внутри выбранной специализации. Обязателен для подачи заявки на вакансию этой специализации. См. [student-actions.md §2.2](student-actions.md#22-самооценка-навыков).

### Группа навыков (Skill Group)
Логический раздел внутри специализации в структуре самооценки. Содержит 2–12 отдельных навыков. В специализации обычно 4–7 групп.

### Навык (Skill)
Конкретная компетенция, которую студент оценивает числом от 1 до 5 (или оставляет пустой).

### Профиль (Profile)
Персональные данные студента: имя, фамилия, email, регион, вуз, «О себе», список выбранных специализаций. Хранится в памяти сессии.

## Статусы и состояния

### Статус проекта
- **Формирование команды** — проект набирает людей, открыты вакансии, задачи и материалы пока не показываются.
- **Работа над прототипом** — команда собрана, идёт активная работа, доступны задачи и ресурсы.

### Состояние вакансии
- **open** — вакансия свободна, студент может подать заявку.
- **submitted** — заявка только что отправлена.
- **applied** — заявка зафиксирована системой.
- **invited** — куратор пригласил студента на эту вакансию.

### Статус задачи
- **Готово** — задача завершена.
- **В работе** — задача выполняется.
- **Не начато** — задача ещё не взята в работу.

### Шкала самооценки
| Значение | Смысл |
|---|---|
| 1 | Нет опыта |
| 2 | Начальный |
| 3 | Средний |
| 4 | Продвинутый |
| 5 | Экспертный |

### Фильтр гипотез
- **Все** — без фильтра.
- **С проектами** — гипотезы, у которых `projects.length > 0`.
- **Без проектов** — гипотезы, для которых ещё не собрались команды.

## Экраны и навигация

### Онбординг (Onboarding)
Серия из 3–4 экранов после первого входа, объясняющая базовые понятия: гипотеза, проект, команда. Показывается один раз за сессию.

### Dashboard (Главный экран)
Основная страница после входа. Показывает проекты, в которых студент участвует, и прогресс по задачам.

### Мои проекты
Блок на главном экране со списком проектов, где студент является участником.

### Каталог проектов / Каталог гипотез
Списковые экраны с фильтрацией. Каталог проектов фильтруется по специализации, каталог гипотез — по наличию проектов.

### Карточка (гипотезы / проекта)
Детальный экран сущности со всей полнотой данных: бриф, команда, вакансии, задачи, материалы.

## Технические термины

### Денормализация
Дублирование полей между сущностями для ускорения отображения списков без дополнительных запросов. Пример — `Project.hypothesisTitle` копирует `Hypothesis.title`.

### Сессионное состояние
Данные, которые живут только в памяти текущей вкладки браузера и сбрасываются при перезагрузке. В прототипе так хранятся: оценки самооценки, переопределения состояний вакансий, флаг участника, прогресс онбординга.

### Header-menu
Верхнее меню-навигация в шапке приложения. Содержит переходы между основными разделами: Главная, Гипотезы, Проекты. Активный пункт выделяется фоном.

### SKILLS_STRUCTURE
Константа в `src/data/skills.ts`, которая описывает полное дерево: специализация → группа навыков → навык. Используется для построения экранов самооценки.

### hypothesisId
Внешний ключ в сущности `Project`, указывающий на родительскую гипотезу.

### isYou
Булев флаг у участника команды, равный `true` для текущего пользователя. Используется, чтобы выделить «себя» в списке членов команды.

### initialState
Поле вакансии, задающее её исходное состояние для текущего пользователя (`open` / `submitted` / `applied` / `invited`).
