# Студент: действия и сценарии

Документ описывает бизнес-логику приложения с точки зрения роли **Студент**. Формат: «что студент видит → что может сделать → какие правила срабатывают».

## 1. Что может делать студент

### 1.1. Знакомство с платформой
- **Онбординг** после первого входа — анкета из двух шагов: «Расскажите о себе» (регион, учебное заведение, увлечения) и «Выбери свою специализацию» (одна из семи). Подробнее — [onboarding.md](onboarding.md).
- **Главный экран (Dashboard)** — показывает проекты, в которых студент уже участвует, и прогресс по задачам.

### 1.2. Работа с гипотезами
- Просматривать список гипотез.
- Фильтровать список: **«Все» / «С проектами» / «Без проектов»**.
- Открывать бриф гипотезы — цель, контекст, задачи, ресурсы, критерии успеха, ожидаемый результат.
- Видеть внутри гипотезы список связанных проектов и переходить в их карточки.

### 1.3. Работа с проектами
- Просматривать каталог проектов.
- Фильтровать проекты по специализации (Backend, Frontend, QA, System Analyst, Product Owner, Designer, Business Analyst).
- Открывать карточку проекта и видеть:
  - описание, куратора, даты, статус;
  - теги (требуемые роли);
  - состав команды (включая отметку «это вы», если студент уже в команде);
  - открытые вакансии;
  - задачи и материалы — **только** если проект в статусе «Работа над прототипом».
- Подавать заявку на вакансию.
- Принимать приглашение от куратора (если вакансия в состоянии `invited`).

### 1.4. Профиль и самооценка
- Редактировать имя, фамилию, email, регион, вуз, краткое «О себе».
- Указывать свои специализации из списка (7 вариантов).
- Проходить самооценку навыков по каждой выбранной специализации: оценка 1–5 («Нет опыта» → «Экспертный») по каждому навыку внутри групп навыков.
- Сбрасывать/обновлять оценки в рамках сессии.

### 1.5. Участие в проекте (если уже в команде)
- Видеть карточку проекта в разделе «Мои проекты» на главном экране.
- Видеть прогресс-бар по задачам.
- Просматривать список задач со статусами: **«Готово» / «В работе» / «Не начато»** и назначенным исполнителем.
- Просматривать ссылки на рабочие материалы (прототипы, датасеты, документы).

## 2. Ключевые сценарии (флоу)

### 2.1. Поиск и подача заявки на проект

```
Логин → Онбординг → Главный экран
      → Вкладка «Проекты»
      → Фильтр по специализации
      → Карточка проекта
      → «Отправить заявку» на вакансии
```

Перед подтверждением заявки система проверяет два условия:

1. **Специализация вакансии должна быть в профиле студента.**
   Если её нет → диалог предлагает добавить специализацию и (опционально) пройти самооценку.
2. **По этой специализации должна быть заполнена самооценка.**
   Если оценка отсутствует → диалог «Пройти сейчас» ведёт в экран самооценки; после завершения флоу возвращает к подаче заявки.

Если оба условия выполнены, студент подтверждает заявку. Вакансия переходит из состояния `open` в `submitted` (далее `applied` после системной обработки).

### 2.2. Самооценка навыков

Запускается из двух точек:
- **Из профиля** — студент сам решает оценить свои навыки.
- **Из флоу подачи заявки** — система требует оценку для выбранной специализации.

Процесс:
1. Выбор специализации.
2. Последовательное прохождение групп навыков (внутри каждой — 2–12 навыков).
3. Оценка каждого навыка по шкале 1–5 или пропуск.
4. Прогресс-бар показывает долю отвеченных.
5. По завершении:
   - из флоу заявки → автоматический возврат к подтверждению;
   - из профиля → сводка по специализации на экране профиля.

### 2.3. Просмотр гипотез и связанных проектов

```
Вкладка «Гипотезы»
  → Фильтр (Все / С проектами / Без проектов)
  → Карточка гипотезы (бриф)
  → Секция «Проекты» внутри гипотезы
  → Переход в полную карточку проекта
```

### 2.4. Участие в активном проекте

Студент уже принят в команду:
- На главном экране появляется блок «Мои проекты» с карточкой и прогрессом.
- В карточке проекта он отмечен флагом `isYou` в списке участников.
- Становятся доступны разделы «Задачи» и «Ресурсы» (только в статусе «Работа над прототипом»).
- Подача новых заявок в этом проекте недоступна.

## 3. Правила доступа и ограничения

| Действие | Условие |
|---|---|
| Подать заявку на вакансию | Специализация вакансии есть в профиле **и** по ней заполнена самооценка |
| Видеть задачи и ресурсы проекта | Проект в статусе «Работа над прототипом» |
| Видеть себя в составе команды | Студент уже принят на вакансию (участник команды) |
| Редактировать чужие данные | Недоступно: студент работает только со своим профилем |
| Создавать проект или гипотезу | Недоступно: студент потребитель контента, не автор |

---

## 4. Что студент **не может** делать

- Создавать и редактировать гипотезы и проекты.
- Управлять вакансиями и составом команды.
- Оценивать других участников.
- Изменять статус проекта или задач.
- Приглашать других студентов.

Эти действия относятся к ролям куратора и администратора, которые пока не реализованы.
